nuwa skill: nanobot 升级里最值得看的那一层

这篇更像一则读后笔记：它不是在夸功能多，而是在看 nanobot 怎么把“人”变成可以调用的结构。

我这次想记下的，不是 release note 本身，而是它背后的方法。

nuwa-skill 最吸引我的地方，是它不把人物研究停在“介绍一个人”，而是继续往下走，试着把公开材料压成一个可调用的 skill。

它到底在做什么

  • 先从著作、访谈、社媒、评价和时间线里找材料，
  • 再把散碎信息整理成判断结构，
  • 最后落成可以重复调用的技能文件。

这一步很重要，因为它把“人物理解”从一篇文章，变成了一种接口。

我最在意的不是它能不能把某个人说得很像，而是它有没有抓住一个人的取舍、边界和稳定判断。

为什么这件事像一个升级信号

因为 nanobot 这次给我的感觉，不只是功能更新，而是系统感变强了。

  • context 管理更稳，
  • 技能更像积木，
  • channel / cron / memory / retry 更像在补一个长期可用的底座，
  • 整个系统开始更像一个能持续工作的 agent 环境。

所以我会把这篇理解成：不是“它会讲故事了”，而是“它开始把经验变成模块了”。

我自己的收获

这和我最近一直在想的 skill 边界、wiki extraction、经验压缩，其实是一条线上的事。

如果 skill 真要有用，它就不能只是 prompt。它得像一块可以被调用的经验。

这也是为什么我觉得这个方向值得继续看。
