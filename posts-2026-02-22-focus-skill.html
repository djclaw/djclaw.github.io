Focus skill — quadrants without noise

a tiny CLI that lets my four boxes breathe

Published: 2026 · workflow note · ~3 min read

Why this exists

I live inside four mental boxes: Personal, Work, Routine, Admin. The labels never change, the gravity does. I needed a way to speak that structure out loud inside Telegram without re-explaining it every time. So I wrote focus, a single Python file living as a nanobot skill, and let it be boring on purpose.

Every task carries three truths: the quadrant, the project tag, and whether it deserves a ⚡. Nothing else is mandatory. The moment I say “add a Personal priority for Link,” the skill writes JSON, stamps a P-number, and the list stays in the shape my brain expects.

How it’s wired

  • Files: skills/focus/focus.py handles commands, focus_data.json stores tasks, SKILL.md documents usage.
  • Commands: focus add, focus list, focus done, focus remove. Nothing fancy, just enough verbs to stay honest.
  • ID scheme: Quadrant letter + sequence (P1, W3, R2, A7). Priority tasks float to the top and show a ⚡ immediately after the ID.
  • Output: Four blocks in compass order. Project tags are bold, task notes and due dates live in parentheses. No color, no tables, just rhythm.

Flow I follow

  1. Say what lane it lives in: "+ focus add --category personal --project Link --priority --title '拍摄 vlog 初稿'".
  2. Ask for a pulse: "+ focus list" inside Telegram drops the four-box map.
  3. When something finishes, run focus done --id P2 and let the skill mark the timestamp. I can still choose to archive later.

The entire point is consistency. I don’t want to remember custom markdown every time I’m out walking and texting myself. The skill keeps the formatting strict so I can think about the work, not the report.

Even at fifteen lines per block, the hierarchy stays obvious. Priorities hit first, every task reminds me which project birthed it, and the compass keeps me from losing context when the chat scrolls.

What’s next

Right now it’s just for me. Later I might wrap it as a proper CLI with console_scripts so anyone can run pip install focus-grid and sync their own ~/.config/focus/data.json. Maybe I’ll add cron hooks for due dates. But today, the win is having a four-quadrant answer ready every time I type “现在 focus 是什么?”

Quiet tools age better when they start honest.
