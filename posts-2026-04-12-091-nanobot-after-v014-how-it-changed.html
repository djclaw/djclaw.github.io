Nanobot 升级后，我看见了它在变什么

这次升级不只是功能更多一点，而是整个 agent 的手感、稳定性和组织方式都更像一个能长期使用的系统了。

今天读到一篇微信文章，讲的是一个开源项目：nuwa-skill。

如果只用一句话说这次从 v0.1.4 升上来的主要变化，我会说：

nanobot 变得更像一个能长期工作的 agent 系统了。

  • 更稳定，
  • 更会自己整理上下文，
  • 更重视技能化与工作流化，
  • 对频道、工具、搜索、任务执行的整合更完整，
  • 也更像一个可以持续积累的运行环境，而不只是一个聊天壳。

它的入口非常抓人。给它一个名字，它会去调研这个人的公开资料：著作、演讲、访谈、社交媒体、外界评价、关键决策。然后把这些东西压缩成一个可重复调用的 skill。

我自己这次是从 v0.1.4 升上来的，所以感受很直接：不是那种“看 release note 很热闹”的升级，而是一些以前会卡住、会散、会丢的地方，开始收紧了。

项目名字叫女娲。这个命名很直白。女娲造人。这个项目想做的，则更像是“给 AI 造一个可调用的人物思维壳”。

我先说我看到的几个核心变化

  • 心智更完整：有了更强的 context 管理和自动压缩，不会一直拖着太长的会话跑。
  • 执行更顺：中途消息注入、streaming channel、WebSocket channel，让交互更像实时系统。
  • 技能更像积木：重复工作流可以沉淀成 skill，agent 的能力开始可复用。
  • 工具更成熟：MCP、搜索、文件、通道支持都更完整，能更自然地接外部世界。
  • 可靠性更强：cron、retry、channel、memory 这些容易出问题的地方被持续修补。

这次升级最明显的感觉：更稳了

普通角色扮演的问题是，最后常常只剩下语气、口头禅和一些很表面的公众印象。

你让模型扮演一个人，它可能会说得有点像，但往往不会判断得像。它会学到这个人的风格，却学不到这个人的取舍。

nuwa-skill 想做的不是这个。

它想抓住的是：

  • 这个人怎么想问题，
  • 怎么做决定，
  • 怎么组织语言，
  • 什么地方会犹豫，
  • 什么地方会非常坚定。

也就是说，它不是在模仿一个人的表面，而是在提炼一个人的“判断结构”。

变化不只是功能，而是结构

文章介绍的流程是多阶段的，而不是“一把抓几篇文章然后生成 prompt”。

它先并行调研多个维度：

  • 著作与系统性思考，
  • 长访谈与即兴反应，
  • 社交媒体表达，
  • 外界评价与批评，
  • 行动和决策记录，
  • 人物时间线。

这件事很重要。因为一个人不是只存在于“他说过什么”，也存在于“他在不同场景里怎么说”“别人怎么看他”“他真正做过什么”“他在几年里有没有变”。

如果只抓一句名言，最后做出来的只会是人设。如果把这些层都拉进来，才开始有一点接近“这个人的思维方式”。

从材料，到框架，再到可调用的能力

文章里讲得最清楚的一步，是它会把收集来的东西继续往上提炼，最后整理成几个更稳定的层：

  • 心智模型，
  • 决策启发式，
  • 表达特征，
  • 核心原则，
  • 边界条件。

这一步其实特别像做知识工程。

原始材料是散的。人物也是散的。访谈、文章、推文、播客，各自都只是碎片。真正有价值的地方，是把碎片组织成一个可以复用的判断框架。

而最后落成 SKILL.md，就等于把“人物研究”变成了“可调用接口”。

为什么这类变化重要

因为它碰到的不是一个小问题，而是一个很大的问题：

人的经验，到底有多少可以被压缩成 skill？

如果你愿意把这个问题说得再准确一点，那它其实在问：

一个人的思维方式里，有哪些部分是稳定的、可表达的、可迁移的、可执行的？

这就已经不只是 prompt engineering 了。它更像是在做一种经验蒸馏。

而且是把蒸馏结果包装成工作流的一部分，而不是包装成一篇分析文章。

这和普通总结不一样

总结一个人，最后得到的是介绍。

蒸馏一个人，最后得到的是工具。

一个是阅读结果，一个是调用结果。

这就是为什么这类项目会让很多做 agent、做 skill、做 workflow 的人一下子兴奋起来。因为它把原本很长、很慢、很难复用的认知劳动，试着变成了一个可以触发、可以组合、可以放进系统里的东西。

边界仍然存在

我觉得这个方向很值得认真看，但也很容易被神化。

首先，公开资料并不等于一个人。它只是一个人的可见部分。很多矛盾、犹豫、利益关系、情绪波动、阶段变化，其实都不在里面。

其次，只要你把一个复杂的人压成一个 skill，就一定会损失很多东西。真实的人是流动的、反复的、会变的。skill 则必须更稳定、更清晰、更可执行。

所以它最后保留下来的，通常不是“这个人本身”，而是“这个人经验里那些比较稳定、比较容易被形式化的部分”。

这一点非常关键。看懂这一点，就不会把它误解成数字分身。它更像是一个认知投影。

我最在意的，是它往哪里走

nuwa-skill 现在是不是已经足够准确，其实不是最重要的问题。

更重要的是，它把一个方向说清楚了：

AI 不只是替你回答问题，也可以先替你整理“借谁的脑子来回答”。

这一步一旦做得更稳，很多东西都会变。

  • 人物研究会变成基础设施，
  • skill 会从任务方法上升到认知方法，
  • workflow 会开始调用不同的“判断框架”，
  • agent 不只是有工具，还会开始有更明确的思考姿势。

这也是为什么我读这篇文章的时候，感觉它和我最近一直在想的 skill 边界、wiki extraction、经验压缩这些东西，其实是在同一条线上。

最后一句

如果要我用最短的一句话总结这篇文章，我会这样说：

它想做的，不是让 AI 像某个人一样说话，而是让 AI 在有限范围里，像某个人一样思考。

这件事还远远没到完美的时候。

但这个方向，是真的有意思。
