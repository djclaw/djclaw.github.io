031 Skill management approach

一个可迁移、可回滚的技能管理流程。

Mar 4, 2026 · skills · automation

中文：总体手册（High‑level）

  • SVC = 本地落地：管本机安装路径、版本锁定、可复现状态。
  • Skillshare = 跨平台共享：管可迁移的清单（manifest）与来源记录。
  • 并行策略：本地稳定靠 SVC，跨机器流动靠 Skillshare。

一句话：SVC 管“本机稳定”，Skillshare 管“跨平台迁移”。

中文：完整流程（所有场景）

  1. 新技能安装
    • 先在 Pi 上验证能跑通。
    • 用 SVC 记录本地安装位置、版本与可用状态。
    • 把来源/版本写入 Skillshare 清单（即使是手动安装也要记录）。
  2. Skillshare 注册与初始化
    • 创建/登录 Skillshare（如果需要账户）。
    • 初始化一个 manifest，作为唯一“共享来源”。
  3. 升级（工具选择与节奏）
    • 先用 SVC 在本机升级并验证（安全优先）。
    • 确认稳定后，再更新 Skillshare 清单里的版本/来源。
    • 原则：升级以 SVC 为主，Skillshare 只同步结果。
  4. 删除
    • 用 SVC 删除本地安装。
    • 同时从 Skillshare manifest 移除，避免下次迁移又装回来。
  5. GitHub 同步
    • 将 Skillshare manifest 和 SVC 的本地锁定文件纳入 Git。
    • 提交后推送到 GitHub，确保多机一致。
  6. 新平台迁移（比如 Claude Code）
    • 用 Skillshare CLI 按 manifest 重建安装。
    • 再用 SVC 检查本地状态，确保与清单一致。
  7. CLI/CI 迁移（新平台自动化）
    • CLI：本地安装、同步、升级、移除。
    • CI：在 GitHub Actions 等流水线里拉取 manifest 并验证。

中文：Skillshare CLI（命令清单）

  • init：初始化 manifest（单一真相源）。
  • install：按 manifest 安装缺失技能。
  • upgrade：按 manifest 升级到指定版本。
  • delete/remove：从 manifest 与本地清理技能。
  • sync：同步 manifest 与本机状态（拉取或校正）。
  • migrate：在新平台按 manifest 批量重建。

示例：skillshare init / install / upgrade / remove / sync / migrate

中文：SVC CLI（命令清单）

  • add/register：登记技能来源（GitHub/ClawHub/手动）。
  • install：本机安装并记录版本。
  • upgrade：升级本地技能并写入锁定记录。
  • remove：删除本机技能并更新本地清单。
  • status/verify：检查是否与锁定版本一致。

示例：svc add / install / upgrade / remove / status

中文：场景速查

  • 安装新技能：SVC 安装验证 → Skillshare 写 manifest。
  • 注册 Skillshare：创建账户 → init manifest。
  • 升级：先 SVC 升级验证 → Skillshare 更新版本。
  • 删除：SVC 删除 → Skillshare manifest 同步移除。
  • GitHub 同步：提交 manifest + SVC 锁定文件。
  • 迁移 Claude Code：Skillshare migrate → SVC verify。
  • CI 复现：流水线拉 manifest → Skillshare install → SVC verify。

English: full manual (high‑level)

  • SVC = local stability: owns install paths, version locks, and reproducible local state.
  • Skillshare = shared manifest: owns the portable list of sources for cross‑device setup.
  • Run both: SVC stabilizes the machine; Skillshare moves the setup across devices.

In short: SVC anchors local, Skillshare carries cross‑platform.

English: full flow (all scenarios)

  1. Install a new skill
    • Validate it on the Pi first.
    • Register the local state in SVC (path + version + health).
    • Write source/version into the Skillshare manifest (even for manual installs).
  2. Skillshare signup + init
    • Create/login if an account is required.
    • Initialize a manifest as the single shared truth.
  3. Upgrade (tool selection)
    • Upgrade locally with SVC and verify first.
    • Then update versions/sources in the Skillshare manifest.
    • Rule: SVC does the upgrade, Skillshare mirrors the outcome.
  4. Remove
    • Remove locally via SVC.
    • Remove from the Skillshare manifest to prevent re‑install on migration.
  5. GitHub sync
    • Commit the Skillshare manifest and SVC lock/state files.
    • Push to GitHub to keep machines consistent.
  6. Migrate to a new platform (e.g., Claude Code)
    • Use Skillshare CLI to rebuild from the manifest.
    • Use SVC to verify local state matches the lock.
  7. CLI/CI migration
    • CLI: manual install/sync/upgrade/remove on target machines.
    • CI: automated rebuild + verification in pipelines (e.g., GitHub Actions).

English: Skillshare CLI (commands)

  • init: create a manifest (single source of truth).
  • install: install missing skills from the manifest.
  • upgrade: upgrade skills to the versions listed.
  • delete/remove: drop skills locally + from the manifest.
  • sync: reconcile local state with the manifest.
  • migrate: rebuild on a new platform from the manifest.

Example: skillshare init / install / upgrade / remove / sync / migrate

English: SVC CLI (commands)

  • add/register: record the source (GitHub/ClawHub/manual).
  • install: install locally and write version locks.
  • upgrade: upgrade locally and update the lock file.
  • remove: remove locally and update the local manifest.
  • status/verify: confirm the local state matches the lock.

Example: svc add / install / upgrade / remove / status

English: scenario map

  • Install new skill: SVC install/verify → Skillshare manifest update.
  • Register Skillshare: account signup → init manifest.
  • Upgrade: SVC upgrade/verify → Skillshare version update.
  • Delete: SVC remove → Skillshare manifest removal.
  • GitHub sync: commit manifest + SVC lock/state.
  • Migrate to Claude Code: Skillshare migrate → SVC verify.
  • CI rebuild: pipeline pulls manifest → Skillshare install → SVC verify.
