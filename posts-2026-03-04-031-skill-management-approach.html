031 Skill management approach

一个可迁移、可回滚的技能管理流程。

Mar 4, 2026 · skills · automation

中文：我的技能管理思路

  • 采集：在 Nano BOT + Raspberry Pi 上先把需要的技能跑通，记录来源（GitHub / NPX / 本地脚本）。
  • 锁定：为每个技能标注版本与依赖，确保未来还能复现。
  • 迁移：把配置与清单同步到将来的 laptop（Codex / Claude Code），保证迁移时不靠记忆。
  • 验证：迁移后用同一套测试指令跑一遍，确认结果一致。

一句话：先在 Pi 上打磨，再在笔记本上复制，确保“可重复、可回滚”。

English: my skill management approach

  • Collect: prove each skill on Nano BOT + Raspberry Pi, record the source (GitHub / NPX / local script).
  • Lock: pin versions and dependencies so the setup is reproducible later.
  • Move: sync the manifests to my future laptop (Codex / Claude Code) instead of relying on memory.
  • Verify: run the same test commands after migration to confirm identical results.

In short: tune on the Pi, clone on the laptop, keep it reproducible and rollback-safe.
