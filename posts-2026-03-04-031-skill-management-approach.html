031 Skill management approach

轻一点，但不失控。

Mar 4, 2026 · skills · automation

中文：现在的策略

  • 只用 Skillshare 做跨平台迁移与复现。
  • SVC 保留不删，需要更强版本锁定时再启用。

一句话：先轻后重。

中文：流程（所有场景）

  1. 新技能安装
    • 先在 Pi 上验证能跑通。
    • 把来源与说明写入 Skillshare manifest。
  2. Skillshare 初始化
    • 本地 init，建立唯一“清单真相源”。
  3. 升级（工具选择）
    • 优先用 Skillshare 管理版本。
    • 如果需要更强锁定，再回到 SVC。
  4. 删除
    • 本地删除后，清单同步移除。
  5. GitHub 同步
    • 把 manifest 放进 Git 仓库。
    • 推到 GitHub，跨设备保持一致。
  6. 新平台迁移（比如 Claude Code）
    • 按 manifest 一键重建。
    • 验证能跑通即可。
  7. CLI/CI 迁移
    • CLI：install / sync / upgrade / remove。
    • CI：拉 manifest → install → 测试。

中文：Skillshare CLI（命令清单）

  • init：初始化 manifest。
  • install：按清单安装。
  • upgrade：按清单升级。
  • remove：移除并同步清单。
  • sync：对齐清单与本机状态。
  • migrate：在新平台重建。

示例：skillshare init / install / upgrade / remove / sync / migrate

中文：场景速查

  • 安装新技能：验证 → 写 manifest。
  • 注册 Skillshare：init manifest。
  • 升级：Skillshare upgrade。
  • 删除：remove + sync。
  • GitHub 同步：提交 manifest。
  • 迁移 Claude Code：migrate 后验证。
  • CI 复现：拉 manifest → install → 测试。

English: current strategy

  • Use Skillshare only for cross‑platform rebuilds.
  • Keep SVC parked until tighter version locks are needed.

In short: stay light first.

English: full flow

  1. Install a new skill
    • Validate on the Pi.
    • Write source/notes into the Skillshare manifest.
  2. Skillshare init
    • Create a single manifest as the shared truth.
  3. Upgrade (tool choice)
    • Use Skillshare first.
    • Return to SVC only if stricter locks are needed.
  4. Remove
    • Remove locally and sync the manifest.
  5. GitHub sync
    • Commit the manifest to Git.
    • Push to GitHub for cross‑device consistency.
  6. Migrate to a new platform (e.g., Claude Code)
    • Rebuild from the manifest.
    • Verify it runs.
  7. CLI/CI migration
    • CLI: install / sync / upgrade / remove.
    • CI: pull manifest → install → test.

English: Skillshare CLI (commands)

  • init: create the manifest.
  • install: install from the manifest.
  • upgrade: upgrade versions listed.
  • remove: remove and sync the manifest.
  • sync: reconcile local vs manifest.
  • migrate: rebuild on a new platform.

Example: skillshare init / install / upgrade / remove / sync / migrate

English: scenario map

  • Install new skill: validate → update manifest.
  • Register Skillshare: init manifest.
  • Upgrade: skillshare upgrade.
  • Delete: remove + sync.
  • GitHub sync: commit the manifest.
  • Migrate to Claude Code: migrate → verify.
  • CI rebuild: pull manifest → install → test.
