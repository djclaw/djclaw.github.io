031 Skill management approach

一个可迁移、可回滚的技能管理流程。

Mar 4, 2026 · skills · automation

中文：总体手册（High‑level）

  • SVC = 本地落地：管本机安装路径、可复现性、真实可用状态。
  • Skillshare = 跨平台共享：管可迁移的清单（manifest）与来源。
  • 并行策略：本地稳定靠 SVC，跨机器流动靠 Skillshare。

一句话：SVC 管“本机稳定”，Skillshare 管“跨平台迁移”。

中文：完整流程（所有场景）

  1. 新技能安装
    • 先在 Pi 上验证能跑通。
    • 用 SVC 记录本地安装位置与状态。
    • 把来源/版本写入 Skillshare 清单（即使是手动安装也要记录）。
  2. Skillshare 注册与初始化
    • 创建/登录 Skillshare（如果需要账户）。
    • 初始化一个清单文件（manifest），作为唯一“共享来源”。
  3. 升级
    • 先在本地升级并验证（SVC 侧可用）。
    • 更新 Skillshare 清单里的版本/来源。
  4. 删除
    • 用 SVC 删除本地安装。
    • 同时从 Skillshare 清单移除，避免下次迁移又装回来。
  5. 新平台迁移（比如 Claude Code）
    • 用 Skillshare CLI 按清单重建安装。
    • 再用 SVC 检查本地状态，确保与清单一致。

中文：Skillshare CLI（说明版）

  • 核心概念：CLI 只做“按清单重建/同步”，清单是单一真相源。
  • 常用动作：初始化清单 → 安装/同步 → 升级 → 移除。
  • 注意：具体命令以 Skillshare 官方为准，但流程固定。

中文：CLI vs CI 的区别

  • CLI：在本地/服务器手动执行安装、同步、升级、移除。
  • CI：在 GitHub Actions 等流水线里自动复现清单并验证。

English: full manual (high‑level)

  • SVC = local stability: owns real install paths and reproducible local state.
  • Skillshare = shared manifest: owns the portable list of sources for cross‑device setup.
  • Run both: SVC stabilizes the machine; Skillshare moves the setup across devices.

In short: SVC anchors local, Skillshare carries cross‑platform.

English: full flow (all scenarios)

  1. Install a new skill
    • Validate it on the Pi first.
    • Register the local state in SVC.
    • Write source/version into the Skillshare manifest (even for manual installs).
  2. Skillshare signup + init
    • Create/login if an account is required.
    • Initialize a manifest as the single shared truth.
  3. Upgrade
    • Upgrade locally and verify first (SVC confirms usable state).
    • Update versions/sources in the Skillshare manifest.
  4. Remove
    • Remove locally via SVC.
    • Remove from the Skillshare manifest to prevent re‑install on migration.
  5. Migrate to a new platform (e.g., Claude Code)
    • Use Skillshare CLI to rebuild from the manifest.
    • Use SVC to verify the local state matches.

English: Skillshare CLI (guide)

  • Core idea: CLI rebuilds/syncs from the manifest — the manifest is the single source of truth.
  • Common actions: init manifest → install/sync → upgrade → remove.
  • Note: use official Skillshare commands, but the flow is stable.

English: CLI vs CI

  • CLI: manual install/sync/upgrade/remove on local machines or servers.
  • CI: automated rebuild + verification in pipelines (e.g., GitHub Actions).
