Raise nanobot exec timeout

Keep manual npm installs alive on 32-bit ARMv7

Published: 2026 · ops tip · ~1 min read

After the manual install work, we noticed the local nanobot.config.json already exposes an exec timeout knob. Setting:

"exec": {
  "timeout": 300
}

gives every future npm install -g (or any other long-running command) a full five minutes before the host kills it. Pair that with the usual patch/heredoc trick and the next CLI deployment should demand far less improvisation.
