039 Google Workspace CLI 快速上手（gws）

2026-03-06

这是一份 gws 的最小命令清单：Gmail、Keep、Tasks、People、Docs、Slides、Calendar，覆盖安装/登录、无头环境配置和最常用的创建与读取动作。

1) 安装与认证（最小流程）

  1. 初始化 + 登录： 
    gws auth setup
gws auth login
  2. 按需选择 Scope（避免测试模式下的 scope 过多）： 
    gws auth login -s gmail,keep,tasks,people,docs,slides,calendar
  3. 无头/CI 机器：在有浏览器的机器完成登录后，把凭据文件复制到服务器并设置环境变量： 
    export GOOGLE_WORKSPACE_CLI_CREDENTIALS_FILE=/path/to/credentials.json
  4. 服务账号：支持使用 GOOGLE_WORKSPACE_CLI_CREDENTIALS_FILE 或 GOOGLE_APPLICATION_CREDENTIALS 指向服务账号凭据文件。

2) 最小命令清单（每个服务 1–2 条）

Gmail

  • 发送邮件 
    gws gmail +send --to alice@example.com --subject 'Hello' --body 'Hi Alice!'
  • 未读邮件速查 
    gws gmail +triage --max 5 --query 'is:unread'

Keep

  • 创建便签 
    gws keep notes create --json '{"title":"Idea","body":{"text":{"text":"Draft outline"}}}'
  • 列出便签 
    gws keep notes list --params '{"pageSize": 5}'

Tasks

  • 列出任务列表 
    gws tasks tasklists list
  • 添加任务 
    gws tasks tasks insert --params '{"tasklist":"TASKLIST_ID"}' --json '{"title":"Follow up"}'

People / Contacts

  • 列出联系人 
    gws people people.connections list --params '{"resourceName":"people/me","personFields":"names,emailAddresses"}'
  • 搜索联系人 
    gws people people.searchContacts --params '{"query":"Ada","readMask":"names,emailAddresses"}'

Docs

  • 创建文档 
    gws docs documents create --json '{"title":"Meeting Notes"}'

Slides

  • 创建演示文稿 
    gws slides presentations create --json '{"title":"Q2 Review"}'

Calendar

  • 列出日历 
    gws calendar calendarList list
  • 创建事件（快捷命令） 
    gws calendar +insert --summary 'Standup' --start '2026-06-17T09:00:00-07:00' --end '2026-06-17T09:30:00-07:00'

3) 把 nanobot 的部分数据迁到 Google？

可以先从联系人（People）与任务（Tasks）开始：把需要提醒、需要跨设备同步的内容放到 Google；其他隐私性更强的内容仍留在本地。

优点

  • 跨设备同步（手机 + Web）更顺畅
  • 生态整合强（Gmail/Calendar/Tasks 互通）
  • gws 提供统一 JSON 接口，自动化更稳

代价

  • 更强的厂商绑定
  • 隐私与合规压力增加（PII 进入云端）
  • API 速率与字段限制

建议：高频提醒/协作的条目上云；长期沉淀、敏感内容保持本地。
