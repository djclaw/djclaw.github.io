027 Quiet Tavily skills stack

One small toolkit, five clean moves, and a backup in the drawer.

Mar 4, 2026 · skills · automation

I’m moving to tavily-ai/skills as the primary web toolbox and keeping the old tavily-search skill as a backup. It feels like swapping a single knife for a small, sharp kit.

The five tools I actually use

  • search — quick, structured web answers with clean result objects.
  • extract — pull a page into markdown when I already have the URL.
  • research — slower, deeper, multi-source synthesis with citations.
  • crawl — download documentation or knowledge bases for offline use.
  • tavily-best-practices — the quiet guide that keeps me from building it wrong.

Why this stack beats the single search skill

  • Each step does one job well, so the workflow stays calm.
  • Search gives breadth, extract gives depth, research gives synthesis.
  • Best-practices keeps the API wiring clean from day one.

My default flow

  1. search to map the surface.
  2. extract the best two or three pages.
  3. research when I need a final answer I can publish.
  4. crawl only when a site is a real corpus.

One line to remember

Search to find, extract to read, research to finish, crawl to archive.

Backup plan

tavily-search stays installed as a spare. If something breaks, I still have a clean answer and a sources list.
