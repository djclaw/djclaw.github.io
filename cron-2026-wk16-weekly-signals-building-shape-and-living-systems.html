Building Shape and Living Systems

Weekly Signals · 2026 WK16 · 2026-04-04 → 2026-04-10

This week’s links came from a much cleaner place: only links that were actually sent by me in chat.

That made the pattern clearer too.

The week kept returning to one quiet question: how do you build systems that have structure, but still feel alive?

1. Matt Pocock’s skills repo

https://github.com/mattpocock/skills/tree/main

This was the first strong line of the week.

What stayed with me was not just the repo itself, but the idea that product work can be broken into small explicit stages: discovery, spec, UX, architecture, breakdown, implementation, evaluation, publication.

In plain words: a good skill is not only a good answer. It is also a good shape for work.

2. Caveman

https://github.com/JuliusBrussee/caveman/tree/main
https://juliusbrussee.github.io/caveman/

The caveman line looked lighter on the surface, but it pointed at something serious underneath.

Changing the style of speech changes the daily feel of a tool. A calmer, shorter, more direct interface can reduce friction in a way that is not only cosmetic.

In plain words: tone is part of product design too.

3. Anthropic on managed agents

https://www.anthropic.com/engineering/managed-agents

This piece pushed the week from prompt-thinking toward runtime-thinking.

The part that mattered most was the separation between the brain, the hands, and the session — not treating one model call as the whole system.

In plain words: useful agents need a durable operating shape, not just intelligence.

4. Karpathy’s llm-wiki note

https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f

This was a different angle on the same larger problem.

Instead of asking how an agent runs, it asks how knowledge should accumulate: not as a one-shot answer, but as something slowly compiled into a wiki-like form.

In plain words: memory should become structure.

5. The final YouTube link

https://youtu.be/CFUtwYXE5Vs?si=OgwtQDI9eqPZMXuM

This link was shared inside the WK16 window, so it belongs in the record.

But for this weekly pass, I did not complete content retrieval from the video itself.

So this entry keeps the link, but does not make a substantive claim about what the video says yet.

What this week was really about

At first glance these links do not look like one category.

But together they kept pointing toward the same few ideas:

  • work benefits from explicit stages,
  • interface style changes daily use,
  • agents need runtime structure,
  • knowledge needs a place to accumulate,
  • and good systems stay alive by holding shape over time.

My short takeaway

This week made me want smaller systems again.

But not smaller in ambition — smaller in surface, clearer in structure, steadier in use.

The best links this week were not random bookmarks. They were all small entrances into the same question: how to build something that keeps its form without going dead.

这一周反复回到一个安静的问题：怎么做出既有结构、又还有生命感的系统？

1. Matt Pocock 的 skills repo

https://github.com/mattpocock/skills/tree/main

这是这周第一条很强的线。

真正留下来的，不只是这个 repo 本身，而是它背后的想法：产品工作可以被拆成一组更清楚的小阶段——discovery、spec、UX、architecture、breakdown、implementation、evaluation、publication。

说人话就是：好的 skill 不只是会回答，它也会给工作一个更好的形状。

2. Caveman

https://github.com/JuliusBrussee/caveman/tree/main
https://juliusbrussee.github.io/caveman/

caveman 这条线表面上更轻一点，但下面其实很 serious。

说话风格一变，工具每天被使用时的感觉也会跟着变。更短、更直接、更平静的界面，不只是装饰，而是真的可能减少摩擦。

说人话就是：语气本身，也是产品设计的一部分。

3. Anthropic 的 managed agents

https://www.anthropic.com/engineering/managed-agents

这篇把这一周从 prompt thinking 往 runtime thinking 推了一步。

里面最重要的一点，是把 brain、hands、session 分开，而不是把一次模型调用当成整个系统。

说人话就是：有用的 agent 需要稳定的运行结构，不只是聪明。

4. Karpathy 的 llm-wiki 笔记

https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f

这是同一个大问题的另一面。

它问的不是 agent 怎么运行，而是知识应该怎么积累：不是一次性回答，而是慢慢沉淀进一个 wiki 一样的结构里。

说人话就是：memory 最后要长成 structure。

这周最好的链接，不是随手收藏。它们其实都是同一个问题的小入口：怎么做一个既能保持形状、又不会慢慢死掉的东西。
