Lego stack: CLI, API, MCP, agents, skills

A short, clean analogy for people just starting out.

Published: 2026 · skills · ~2 min read

The Lego picture

Think in bricks and builds. Keep it light. Keep it useful.

  • CLI / API = raw bricks. Powerful, but you need to know the shapes.
  • MCP = the brick standard. Same studs, same fit, everywhere.
  • Skill = a small build. A finished piece you can reuse.
  • Agent = the builder. Chooses bricks, snaps them together.

Why it matters

If it’s just you and a fixed flow, grab the bricks and build fast.

If it’s a shared tool, MCP makes the bricks consistent so everyone can build.

One clean example

You want a weekly GitHub report.

  • CLI/API bricks: search issues, pull PRs, post a summary.
  • MCP: the same calls, but standardized and shareable.
  • Skill: the finished report flow, ready to run.
  • Agent: decides when to run it and what to include.

Small bricks. Clean builds. No clutter.

中文版

轻一点，简单一点。用乐高去想就行。

  • CLI / API：原始砖块。能用，但你得记住形状和用法。
  • MCP：统一的砖块标准。接口一致，所有砖都能拼。
  • Skill：拼好的小模块。拿来就用。
  • Agent：搭建的人。选砖，组合，决定流程。

为什么有用

只有你自己用、流程固定：直接拿 CLI/API 最快。

多人共用、需要治理：MCP 让砖块统一，扩展更稳。

一个小例子

你要做每周 GitHub 报表。

  • CLI/API 砖：查 issue、拉 PR、发摘要。
  • MCP：同样的调用，但标准化、可共享。
  • Skill：固定的报表流程。
  • Agent：决定什么时候跑、包含哪些内容。

砖块小，构建干净，就会稳定。
