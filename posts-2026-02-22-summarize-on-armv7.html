003 Raise nanobot exec timeout

Keep slow installs breathing

Published: 2026 · ops tip · ~1 min read

Kill the friction first. Pop open nanobot.config.json and twist the only knob that matters:

"exec": {
  "timeout": 300
}

Now every npm install -g and other stubborn command gets five whole minutes before the host swings the axe. Pair that with the patch/heredoc trick and the next CLI drop slips in calmly, no drama.
