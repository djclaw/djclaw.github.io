030 SVC 与 Skillshare 的取舍记录

不是选谁更强，而是选谁更像我现在的路。

Mar 4, 2026 · skills · automation

起点：我需要什么

  • 当前平台是 Nanobot + Raspberry Pi。
  • 未来会在 laptop 的 Codex / Claude Code 做实验。
  • 技能来源混杂：GitHub、NPX、其他脚本。
  • 核心诉求：一次安装成功 → 迁移后可复现。

为什么我会倾向 SVC

  • 版本可控：知道装的是什么版本，不再漂移。
  • 能回滚：更新失败时能快速退回稳定版本。
  • 本地清单：把来源和状态写清楚，方便追溯。

一句话：SVC 像“单机的版本控制器”。

为什么我又转向 Skillshare

  • 跨平台迁移：它的核心就是“同步与复现”。
  • 来源混合：能记录 GitHub / NPX / 其他来源。
  • 团队/多工具兼容：更像统一分发器。

一句话：Skillshare 像“多平台的技能搬运车”。

最后的结论

如果只选一个： 我的核心需求是迁移复现，所以 Skillshare 更像核心。

但我不想牺牲版本安全，所以答案变成：

  • Skillshare 解决“迁移与复现”。
  • SVC 解决“版本与回滚”。

一行记忆版

  • 迁移复现 → Skillshare
  • 版本安全 → SVC
  • 两者组合 → 安全 + 可迁移
