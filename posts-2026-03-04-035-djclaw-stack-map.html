035 DJClaw 栈地图：目录、调用链与启动命令

2026-03-04

补一份清晰的“栈地图”：文件放哪、谁调用谁、结构长什么样、怎么启动。插件先留占位，等需要时再填充。

结论（先读 30 秒）

  • 目录关系先固定。核心放 src/，skills 放 skills/，插件是可选壳。
  • 调用链从 CLI 开始。cli → mcp server → skills，插件只是包装入口。
  • 启动命令一条线。先 MCP，再看需不需要插件。

调用链（从入口到执行）

cli.py
  └─ mcp/server.py
      └─ skills/*/SKILL.md + scripts

optional:
  plugins/*_wrapper.py
    └─ calls cli.py or mcp/server.py via stdio

结构 + 启动命令

  • MCP 主线：djclaw mcp --config ~/.djclaw/config.yaml
  • 插件占位：djclaw plugin --target claude --stdio（仅包装，不是核心）

插件占位模板（可选）

  • 目录：src/djclaw/plugins/
  • 职责：负责把平台协议转成 MCP/CLI 调用。
  • 开关策略：有分发需求再开，不要影响核心。

CLI vs 脚本（含 CI 关系）

  • Python 脚本可以替代 CLI。只要运行环境能执行脚本，它就能充当入口。
  • CLI 更通用：语言无关、容易被调用/隔离、利于跨环境。
  • CI 更偏好 CLI：命令稳定、易于流水线编排和复现。

CLI vs Script (and CI)

  • A Python script can replace a CLI as long as the environment can execute it.
  • CLI stays more universal: language-agnostic, easier to invoke/sandbox across environments.
  • CI prefers CLI: stable commands and reproducible pipelines.

English Version

A compact stack map: where files live, who calls whom, the structure, and the boot command. Plugin wrappers are placeholders only.

Quick take

  • Freeze folder layout.Core in src/, skills in skills/, plugins are optional shells.
  • Call chain starts at CLI.cli → mcp server → skills. Plugins only wrap.
  • One boot path.MCP first, plugins only if you need distribution.
