025 Two skill repos, quiet gains

两个仓库，六个技能。先记录，再变强。

Mar 3, 2026 · skills · automation

中文

这次我只装了两个仓库，但实际得到 6 个技能。一半是“记忆与习惯”，一半是“如何创建技能”。我想知道它们到底能不能让 nanobot 变得更会做事。

Repo 1: humanplane/homunculus

安装到的技能：session-memory、instinct-apply

1) session-memory

用途：跨会话记忆 + 自动观察器

怎么工作：

  • 启动时加载身份与历史（例如 .claude/homunculus/identity.json）。
  • 如果有观察记录，会后台启动 observer。
  • observer 把反复出现的行为提炼为“instinct”。

典型用例：多天持续开发同一项目；希望 AI 记住你的命名/提交/测试习惯。

2) instinct-apply

用途：在你开始工作时提醒已学会的习惯

怎么工作：

  • 读取个人与继承的 instinct 文件。
  • 按触发条件提示建议。
  • 每条 instinct 有信心等级，用于提醒强度。

典型用例：你有固定的测试/提交流程，不想每次重复解释。

Repo 2: GBSOSS/skill-from-masters

安装到的技能：skill-from-masters、search-skill、skill-from-github、skill-from-notebook

1) skill-from-masters

用途：从“实战高手”方法里生成技能

工作流：

  1. 收窄问题范围。
  2. 找黄金案例与失败案例。
  3. 归纳有效/无效模式。
  4. 必要时用理论解释。
  5. 交给 skill-creator 输出技能。

适合：你要认真做一个可长期复用的技能。

2) search-skill

用途：从可信来源找现成技能

怎么工作：先官方再社区，过滤质量与更新，避免“野生技能”。

适合：你不想重复造轮子。

3) skill-from-github

用途：从高质量 GitHub 项目提炼方法，再生成技能

流程：搜索 → 用户确认 → 深度阅读 → 方法抽取 → 生成技能。

适合：你想把成熟项目的思路变成可调用的流程。

4) skill-from-notebook

用途：把文章/教程/案例转成技能

两条路：教程型提取步骤；成品型逆向方法。

适合：你读到很好的资料，想一键复用。

我建议的 nanobot 轻量配置

  • 先让 homunculus 跑一段时间。它需要真实任务来形成稳定 instinct。
  • 只有在要做“正式技能”时才触发 skill-from-masters。它很重，但产出也重。
  • 做技能前先问三个问题：
    • 有没有现成技能？→ search-skill
    • 有没有成熟项目？→ skill-from-github
    • 有没有好文章？→ skill-from-notebook

English

I installed two repos and got six skills. Half are about memory and habits, half are about creating new skills. Here’s what they do and how I’m setting up nanobot around them.

Repo 1: humanplane/homunculus

Skills installed: session-memory, instinct-apply

1) session-memory

Purpose: cross-session memory + background observer

How it works:

  • Loads identity/history on start (e.g., .claude/homunculus/identity.json).
  • Spawns an observer if observations exist.
  • Turns repeated behavior into small “instinct” rules.

Use cases: multi-day work on one project; you want the AI to remember your style.

2) instinct-apply

Purpose: surface learned habits when you start working

How it works:

  • Reads personal/inherited instinct files.
  • Triggers suggestions based on conditions.
  • Each instinct has a confidence score.

Use cases: stable testing/commit routines you don’t want to repeat.

Repo 2: GBSOSS/skill-from-masters

Skills installed: skill-from-masters, search-skill, skill-from-github, skill-from-notebook

1) skill-from-masters

Purpose: build skills from proven expert practice

Flow:

  1. Narrow the task.
  2. Find golden cases and failures.
  3. Extract what works vs. what doesn’t.
  4. Explain with theory if needed.
  5. Hand off to skill-creator.

2) search-skill

Purpose: find existing skills from trusted sources

How it works: prioritize official/curated sources and filter quality/recency.

3) skill-from-github

Purpose: learn from a high-quality GitHub project, then create a skill

Flow: search → confirm → deep read → method extraction → generate skill.

4) skill-from-notebook

Purpose: convert articles/tutorials/examples into reusable skills

Two paths: tutorial → extract steps; finished work → reverse engineer method.

My lightweight setup

  • Let homunculus run for a while. It needs real tasks to form stable instincts.
  • Trigger skill-from-masters only for serious skills. It’s heavy, but the output is too.
  • Before making a skill, ask:
    • Is there an existing skill? → search-skill
    • Is there a strong project? → skill-from-github
    • Is there a great article? → skill-from-notebook
