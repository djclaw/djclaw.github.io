037 Skill-from-masters 简介

2026-03-04

1. skill-from-masters 是什么

  • 它是一个“先学方法论再生成 skill”的元技能框架。
  • 核心思想：先找专家方法/高质量案例/反例，再交给 skill-creator 生成最终可执行 skill。
  • 不是工具包装器，而是“知识提炼 + 结构化落地”。

2. 它包含什么（仓库实际内容）

  1. 主 skill：skill-from-masters/SKILL.md
  2. 辅助 skill：
    • skills/search-skill/SKILL.md（先找现成 skill）
    • skills/skill-from-github/SKILL.md（从优质 GitHub 项目学习后生成）
    • skills/skill-from-notebook/SKILL.md（从文档/范例反推方法）
  3. 参考知识库：
    • skill-from-masters/references/methodology-database.md
    • skill-from-masters/references/skill-taxonomy.md

3. 输入/输出模型（你最关心的）

  1. 主路径 skill-from-masters
    • Input：目标 skill 主题 + 场景约束（例如“做一个 Linko 学习计划 agent，面向 PM”）
    • Output：候选专家框架、黄金案例、反模式、最终可交给 skill-creator 的结构化要点
  2. skill-from-github
    • Input：任务目标 + 技术约束
    • Output：候选 GitHub 项目 → 选定项目的“可迁移方法” → 新 skill 草案
  3. skill-from-notebook
    • Input：文档/URL/视频/成品范例
    • Output：提取或逆向出的步骤、决策点、质量标准、常见错误 → 新 skill 草案
  4. search-skill
    • Input：能力需求描述
    • Output：现有 skill 备选清单（可先复用，减少重复造轮子）

4. 对 Linko 的几条可行使用路径（推荐顺序）

  1. 路径 A（最快）：search-skill 先找现成 → skill-from-masters 做差异增强
    • 适合：快速产出第一版 Linko agents/skills
  2. 路径 B（质量优先）：skill-from-masters 主路径
    • 输入：Linko 业务场景（学习计划/资源筛选/内容写作）
    • 输出：每个 agent/skill 的方法论骨架（steps、pitfalls、quality checks）
  3. 路径 C（工程化）：skill-from-github
    • 适合：从成熟开源项目借鉴“工作流设计、评分、排序、结构化输出”
  4. 路径 D（内容资产化）：skill-from-notebook
    • 适合：把已有 Linko 文档、优秀笔记、历史对话沉淀成可执行 skill

5. 面向当前目标（生成 Linko agents + skills）的 WS2 建议

  1. 先定 4 个目标资产：orchestrator / study-planner / resource-curator / content-writer。
  2. 每个资产先走 skill-from-masters，产出：
    • do/don’t
    • 决策条件
    • 输出格式（summary/evidence/next_action）
  3. 再用 skill-creator 统一生成 SKILL.md / agent md。
  4. 最后做一次“Linko MCP 对齐检查”：只允许调用现有 3 个 tools。

6. 一个重要提醒（推断）

  • 仓库 README 与实际 repo 组织有轻微差异（例如 README 文案里的安装示例和当前 repo 名称/结构）。
  • WS2 实施时以“当前仓库 tree + SKILL.md 原文”为准，而不是只看 README 文案。
