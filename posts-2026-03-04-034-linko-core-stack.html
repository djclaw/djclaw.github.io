034 Linko 核心栈

2026-03-04

Linko 想要一条能跑在 nanobot / codex / CI 的主线。核心很简单：MCP + skills + 多角色 agent。先跑起来，再包装。

要点

  • 核心用 MCP + skills。可执行、可复用、可测试。
  • 插件只是外壳。平台分发，不是核心能力。
  • 推荐混合包。一个 Python 核心 + 一个可选 skills 包。

为什么不从插件开始

  • 插件是平台语法。
  • 在 CI 里跑不顺。
  • 先把核心跑通，再谈分发。

仓库骨架

repo/
  pyproject.toml
  README.md
  src/
    linko/
      cli.py
      mcp/
        server.py
      plugin/
        wrapper.py
      config/
        default.yaml
  skills/
    <skill_name>/
      SKILL.md
      scripts/
      assets/

CLI 入口

  • MCP：linko mcp --config ~/.linko/config.yaml
  • 插件壳：linko plugin --stdio

给 Codex 的任务清单

  1. 生成 repo 结构
  2. 实现 CLI（mcp / plugin）
  3. 搭 MCP server 骨架
  4. 做一个最小可运行 skill
  5. 补 README（安装 / 运行 / skills 路径）
  6. 可选：plugin wrapper

一句话

Linko 先把 MCP + skills 跑顺，再加分发外壳。

English Version

Linko needs a core that runs everywhere: MCP + skills + multi‑role agents. Ship the core first. Wrap later.

Key calls

  • MCP + skills are the core.Runnable and testable.
  • Plugins are packaging.Distribution only.
  • Ship a hybrid.Python core + optional skills pack.
