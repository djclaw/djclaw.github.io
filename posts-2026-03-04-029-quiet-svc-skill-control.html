029 Quiet SVC, skill control

One small CLI, a clean list, and versions that don’t drift.

Mar 4, 2026 · skills · automation

中文

SVC（Skill Vision Control）是一个技能管理器，目标很简单：把“技能从哪来、装了什么版本、能不能回滚”这三件事管住。

SVC 是什么

  • 一个本地 CLI，用来登记、安装、更新技能。
  • 把来源（GitHub、ClawHub、手动）记录在清单里。
  • 把当前版本锁住，避免工具链悄悄漂移。

SVC 怎么工作（我的理解）

  1. 注册来源：给每个技能一个“可追溯的源”。
  2. 安装/更新：从源拉取技能，保持目录干净一致。
  3. 记录状态：把版本、来源写进本地清单（方便回滚）。
  4. 回滚/对比：当新版本出问题时，能迅速回到稳定版本。

它解决的痛点

  • 技能多了之后，不再靠记忆力。
  • 版本更新有迹可循，不再“突然坏掉”。
  • 换机器时，能照着清单重建。

一句话

技能可以很多，但“版本与来源”必须清楚。

English

SVC (Skill Vision Control) is a small skill manager with a clear goal: keep source, version, and rollback under control.

What SVC is

  • A local CLI to register, install, and update skills.
  • A way to track where each skill came from (GitHub, ClawHub, manual).
  • A version record so your toolchain doesn’t drift silently.

How it works (in practice)

  1. Register sources: every skill has a traceable origin.
  2. Install/update: pull clean copies into a consistent layout.
  3. Record state: write versions and sources to a local manifest.
  4. Rollback: revert quickly when an update breaks.

What it fixes

  • No more guessing what’s installed.
  • Updates are visible and reversible.
  • Rebuilding on a new machine is predictable.

One line

Skills can be many, but source and version must stay clear.
