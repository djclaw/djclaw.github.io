006 Claude Skills 完整指南（XL 中文总结）

基于 Anthropic 官方 PDF《The Complete Guide to Building Skills for Claude》的超长中文总结，适合快速理解“技能”体系的结构、设计方法与落地流程。

Published: 2026 · skills · ~8 min read

1) Skills 的核心概念

Skills 是“可打包的任务说明书”。它把一组可复用的工作流、工具调用方式、错误处理、输出格式等内容，封装成固定结构，交给 Claude 在需要时自动加载并执行。

技能的目标不是让模型更聪明，而是让流程更稳定：减少反复解释、提升一致性、降低错误率。

2) 目录结构与渐进披露（Progressive Disclosure）

一个 skill 至少包含 SKILL.md，可选包含 scripts/、references/、assets/ 等目录。加载分三层：

  • 第一层：YAML frontmatter（始终加载）——决定“什么时候该用这个技能”。
  • 第二层：SKILL.md 正文（按需加载）——给出操作步骤、判断规则与输出要求。
  • 第三层：链接文件（再按需加载）——更多细节、素材或模板。

目的：减少 token 浪费，让模型只在必要时读取更深的内容。

3) 技能与 MCP 的关系

MCP 负责“连工具”，skill 负责“教怎么用”。

  • MCP 提供连接、权限和工具清单。
  • Skill 提供流程、规则与最佳实践。

4) 三类典型用例

  1. 文档与资产生成：输出更稳定的文案、报告、代码、PPT 等。
  2. 流程自动化：多步骤工作流（含校验与回退）。
  3. MCP 强化：为特定工具组合提供“固定打法”。

5) 评估成功的标准

  • 触发准确率：该触发时能触发，不该触发时保持安静。
  • 工具调用效率：调用正确、顺序合理、少走弯路。
  • 用户负担降低：用户不必重复解释。
  • 输出一致性：在不同会话/时间点仍能保持风格与结构一致。

6) 技术规范与硬性要求

  • 文件名必须是 SKILL.md。
  • 目录与 name 均为 kebab-case（短横线命名）。
  • YAML frontmatter 必填：name + description（描述需明确触发条件，且 < 1024 字符）。
  • 安全与合规：frontmatter 禁止 XML 角括号，名称禁止保留字（如 claude / anthropic）。
  • 指令必须可执行：避免抽象描述，必须让模型“知道怎么做”。

7) 测试与迭代建议

建议进行三层测试：

  • 触发测试：能否在正确场景激活，并忽略错误场景。
  • 功能测试：输出是否正确、调用是否成功。
  • 性能对比：有无 skill 的情况下差异是否明显。

官方建议使用 skill-creator 工具来生成、评审与修订技能。

8) 分发方式与组织级部署

  • 个人：可通过上传 ZIP 到 Claude.ai 设置。
  • 团队/组织：管理员可将 skill 部署到工作区。
  • API：支持通过 /v1/skills 接口管理。

9) 常见模式（Patterns）

  • 顺序编排：严格步骤执行（适合有依赖的流程）。
  • 多 MCP 协作：跨系统工具链（如 Figma → GitHub）。
  • 迭代优化：多轮修正直到达到质量阈值。
  • 上下文选择：根据文件类型/大小选工具。
  • 领域增强：加上行业特定规则（如金融合规）。

10) 常见问题与排查点

  • 上传失败：多数是文件命名或 YAML 格式问题。
  • 触发不足：description 太泛，缺少明确触发语。
  • 模型偷懒：需要加明确要求和校验步骤。
  • 输出漂移：需要明确结构模板与“不可省略项”。
