036 Command / Skill / Script / Hook

2026-03-04

一些开发 plugin 的总结，尽量保留原话，整理成更清晰的结构。

核心区别：固定程度 + 触发方式

command

  • 显式触发（用户主动调用某个命令）
  • 流程更固定，像一个标准操作手册
  • 适合“我现在就要做这件事”的快速入口（例如 study-plan）

skill

  • 隐式触发（模型判断当前任务该用哪个 skill）
  • 更偏能力模块，适应范围更广
  • 适合“根据上下文自动选策略”的场景（例如 notes/resources/subjects 的组合调用）

一句话：

  • command = 固定工作流入口
  • skill = 可复用能力单元

在项目里的推荐分工

  • 高频场景给 commands/（稳定、可预期）
  • 复杂推理和组合调用放 skills/（灵活、可扩展）

script 与 hook 的定位

scripts/

  • 可执行小工具（bash/node/python）
  • 被 commands 或 hooks 调用
  • 你能单独运行、单独测试，确定性强

hooks/

  • 宿主事件触发（自动执行）
  • 例如：用户发消息后、tool 调用后、会话结束时
  • 自动化强，但风险和调试复杂度更高

CLI 与 scripts 不一样

  • CLI：给用户跑的安装/诊断入口（list/install/doctor）
  • scripts：给 command/hook 调用的小执行单元（低层工具）

skills/commands 不建议直接调用 CLI。可以调用 scripts（如果宿主支持执行 shell/脚本）。

CLI 主要用于“安装和运维”，不是运行时工作流。
