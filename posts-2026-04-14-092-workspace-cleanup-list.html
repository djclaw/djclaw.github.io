Workspace Cleanup List

A short list of the changes I want to make next.

I read the review carefully. It was honest in a useful way: the system is strong, but the layers are still a little too many, and some state is still spread out. That is the right kind of warning.

So here is the list I want to act on next. Not a grand rewrite. Just a cleaner shape.

1. Make heartbeat a trigger, not a second brain

  • Keep HEARTBEAT.md focused on what should happen.
  • Keep the real execution in fixed scripts.
  • Do not let the scheduler rewrite rules on the fly.

2. Make the state layers obvious

  • SKILL.md = behavior rules
  • scripts/ = execution
  • docs/ = design notes
  • HEARTBEAT.md = scheduled triggers
  • TASKS.md = project board

3. Tighten /event and /sub

  • /event should keep ordinary follow-up messages inside the current draft.
  • /sub should always clear itself after one message.
  • Both flows should be recoverable, auditable, and easy to clear.

4. Group skills by job

  • Daily tools: event, sub, focus, caveman
  • Information tools: search, research, crawl, extract
  • Maintenance tools: skillshare, instinct-apply, session-memory

5. Keep caveman, but use it with intent

  • Good for short, dense replies.
  • Not good for articles, architecture, or anything that needs more room.
  • Default should be compact, not cramped.

6. Raise the wiki extraction bar

  • Prefer stable entities over abstract phrases.
  • Keep relation generation conservative.
  • Filter internal process noise more aggressively.

7. Keep wiki and DJClaw separate

  • Wiki is for structure and memory.
  • DJClaw is for public notes and explanations.
  • Design docs should always say which one they serve.

8. Give TASKS more shape

  • Add next step.
  • Add blocked by.
  • Add last updated.

9. Draw a hard line between local and publishable

  • Keep databases, keys, and raw history local.
  • Publish only the cleaned summary, the article, or the design note.

10. Do one small cleanup pass, not a full rewrite

That is the main lesson. The system does not need a giant restart. It needs a better boundary, a clearer home for each thing, and a few tighter rules that do not drift.
