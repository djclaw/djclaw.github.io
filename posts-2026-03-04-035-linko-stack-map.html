035 Linko 栈地图

2026-03-04

这份栈地图只回答三件事：文件放哪、谁调用谁、怎么启动。插件只是占位，不是主线。

快读版

  • 目录先定型。核心在 src/，skills 在 skills/，插件可选。
  • 调用链从 CLI 开始。cli → mcp server → skills。
  • 启动命令一条线。MCP 先跑，插件后补。

文件摆放（简化）

repo/
  pyproject.toml
  README.md
  src/
    linko/
      cli.py
      mcp/
        server.py
      config/
        default.yaml
      plugins/             # optional placeholders
        claude_wrapper.py
        openai_wrapper.py
  skills/
    <skill_name>/
      SKILL.md
      scripts/
      assets/

调用链

cli.py
  └─ mcp/server.py
      └─ skills/*/SKILL.md + scripts

optional:
  plugins/*_wrapper.py
    └─ calls cli.py or mcp/server.py via stdio

结构 + 启动命令

  • MCP 主线：linko mcp --config ~/.linko/config.yaml
  • 插件占位：linko plugin --target claude --stdio

插件占位说明

  • 目录：src/linko/plugins/
  • 职责：平台协议 → MCP/CLI 调用。
  • 开关策略：有分发需求再开。

English Version

A compact stack map for Linko: folders, call chain, and boot path. Plugins are placeholders only.

Quick take

  • Freeze layout.Core in src/, skills in skills/, plugins optional.
  • Call chain starts at CLI.cli → mcp server → skills.
  • One boot path.MCP first, plugins later.
