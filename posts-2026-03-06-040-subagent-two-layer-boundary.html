040 Subagent 的两层边界（036 续篇）

2026-03-06

这是一篇 036 的续篇：把 subagent-first 的触发逻辑讲清楚，再把“两层 vs 三层”说人话，并说明为什么本项目暂时不允许三层。

触发逻辑：什么时候会叫 subagent

  • 探索 / 调研 / 对比 / 收集：默认交给 subagent
  • 直接执行 / 修改 / 落地：主控直接做
  • 用户明确要求主控执行：主控亲自做

   |     |-- 直接执行/修改 → Main Agent
   |
   v

   |
   v

   |
   v

   |
   |-- 探索类 → Subagent
   |
   v
Subagent
   |
   |-- (禁用) 再拆成 SubSubagent
   v
Main Agent → 汇总

两层 vs 三层（说人话）

  • 两层：稳、快、便宜、好维护。主控只整合一个分身的结果。
  • 三层：能细分更多支线，但成本高、容易失控、调试难。

为什么本项目暂时不允许三层

  • 树莓派环境：多层会明显变慢，失败率更高。
  • 输出导向：DJClaw 更需要稳定、可发布的结果。
  • 可控性：两层足够拆大部分研究任务。

如果以后要开三层，需要什么条件

  • 明确的“多线研究”需求
  • 允许 subagent 再 spawn 子分身的机制
  • 成本与稳定性预算

一句话：两层是现在的甜区。三层不是不能，是现在不值得。

English (plain language)

This is a follow‑up to 036. It explains when subagents are triggered, shows the two‑layer flow, and why this project stays two‑layer for now.

When a subagent is triggered

  • Exploration / research / comparison / collection: default to a subagent
  • Direct execution / edits: main agent handles it
  • User explicitly asks main agent: main agent does it

Current diagram (two layers)

   v
Main Agent

   |-- classify task
   |     |-- explore/research/compare → Subagent
   |     |-- execute/edit → Main Agent
   |
   v
Subagent
   |
   v
Main Agent → final response/action

Three-layer idea (disabled)

User Request
   |
   v
Main Agent
   |
   |-- explore → Subagent
   |
   v
Subagent
   |
   |-- (disabled) spawn SubSubagent
   v
Main Agent → summary

Two layers vs three layers

  • Two layers: stable, fast, cheaper, easier to maintain.
  • Three layers: deeper branching, but higher cost and harder to debug.

Why this project stays at two layers (for now)

  • Raspberry Pi constraints: more layers = slower and less stable.
  • Publishable output: we value clean, reliable results.
  • Control: two layers is enough for most tasks.

What would be required to allow three layers later

  • A clear need for multi‑track research
  • Framework support for sub‑subagents
  • Budget for cost and stability risk

One line: two layers is the sweet spot right now. Three layers is possible later, but not worth it today.
